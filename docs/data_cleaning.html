DATA CLEANING

En este documento se procede con el desarrollo del EDA alineado con el modelo de Munzner. Aqui se valida fundamentalmente la consistencia de las variables dsde el punto de vista fisico. Por ejemplo, que la generacion solar sea cero durante la noche y que la generacion eolica no tenga valores negativos fisicamente imposibles. Ademas se analiza la pertinencia de crear nuevas variables que permitan llegar a una visualizacion clara y legible de patrones temporales y estacionalidad.

Justificacion de las decisiones tomadas.

Ademas del procedimiento general de un EDA tipico como el analisis granularidad, de completitud y duplicidad, este dataset requiere deuna ingenieria de variables que involucra el analisis de consistencia fisica, la creacion de una nuevas variables, reduccion de variables redundantes, analisis de variables significativas. La pregunta principal incluye el analisis de patrones temporales y estacionalidad de la generacion combinada entre generacion eolica y solar; este reequerimiento obliga a tener una variable que muestre la “total_production”. El dataset tiene originalmente 43 variables, 8 de las cuales se detecto que son redundantes mediante criterios fisicos. El criterio de alta correlacion de variables se reafino al complementarlo con el criterio de redundancia para disminir el ruido en el dataset. Finalmente, el dataset resultante “df_clean” contiene lasvariables de importancia y consistentes para la generacion de visualizaciones en la siguiente fase del proyecto.
